[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture of the system.
Our workflow begins at the Security Checkpoint, which intercepts all queries to scrub PII and check for prompt injection before any LLM is called.
If it passes, the query reaches the Orchestrator Node, which runs our main coordinator agent.
The Orchestrator Agent analyzes the query and delegates tasks to our specialized sub-agents: the Medication Assistant, the Appointment Coordinator, or the Symptom Tracker.
These sub-agents communicate directly with our local CareSync MCP Server to query and update the patient's records.
If a patient requests to schedule or change a clinical appointment, the Orchestrator routes the flow to the Human Approval Node.
This node triggers a human-in-the-loop interruption, pausing the execution until the user manually confirms the action, after which the workflow resumes.
Finally, the results are collated and returned via the Final Output Node.

[3:00 — SECURITY & MCP]
Our security checkpoint uses custom regex pattern matching to redact sensitive emails or MRNs, blocks prompt injection, and writes structured JSON audit logs for absolute traceability.
Our custom MCP server exposes secure tools like get_patient_medications and log_patient_symptom, cleanly separating patient database operations from the model's reasoning loop.
